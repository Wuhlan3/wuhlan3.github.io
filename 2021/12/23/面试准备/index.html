语言：C++

1. C++的四种智能指针

我们为什么需要使用智能指针呢？其作用就是用来管理一个指针，避免咱们程序员申请的空间在函数结束时忘记释放，造成内存泄漏的情况发生。智能指针的本质是一个类，当超出类的作用域时，类就会自动调用析构函数，自动释放资源。解决了什么问题：空指针与野指针；对象重复释放的问题；内存泄露的问题；new后忘记delete的问题

C++11 中封装在了 <memory> 头文件中。四种智能指针分别是：

  • auto_ptr（C98方案，C11已经弃用）采用所有权模式

  • unique_ptr（替代auto_ptr）：“独占”所指的对象。

  • shared_ptr（共享型，强引用）：允许多个指针指向同一个对象。使用计数机制来表明资源被几个指针共享。可以使用函数use_count()来查看对象的所有者。当调用release()时，指针回释放资源所有权，计数减一。当计数为0时，资源会被释放。

  • weak_ptr（弱引用）：指向shared_ptr所管理的对象。不控制对象生命周期，即不能分配与释放对象的内存。它可以由一个shared_ptr或另一个weak_ptr对象来构造，它的构造和析构不会引起计数的变化。

    weak_ptr 是用来解决shared_ptr相互引用时造成的死锁问题，如果两个shared_ptr相互引用，那么两个指针的引用计数永远不会下降为0。

    weak_ptr 和shared_ptr 之间可以相互转化：shared_ptr可以直接赋值给weak_ptr，weak_ptr可以调用lock()函数来转换为shared_ptr

2. C++的内存管理

C++ 内存分区：栈、堆、全局/静态存储区、常量存储区、代码区。

栈：存放函数的局部变量、函数参数、返回地址等，由编译器自动分配和释放。
堆：动态申请的内存空间，就是由 malloc 分配的内存块，由程序员控制它的分配和释放，如果程序执行结束还没有释放，操作系统会自动回收。
全局区/静态存储区（.bss 段和 .data 段）：存放全局变量和静态变量，程序运行结束操作系统自动释放，在 C 语言中，未初始化的放在 .bss 段中，初始化的放在 .data 段中，C++ 中不再区分了。
常量存储区（.data 段）：存放的是常量，不允许修改，程序运行结束自动释放。
代码区（.text 段）：存放代码，不允许修改，但可以执行。编译后的二进制文件存放在这里。
说明：

从操作系统的本身来讲，以上存储区在内存中的分布是如下形式(从低地址到高地址)：.text 段 --> .data 段 --> .bss 段 --> 堆 --> unused --> 栈 --> env

#include <iostream>
using namespace std;

int g_var = 0; // g_var 在全局区（.data 段）
char *gp_var;  // gp_var 在全局区（.bss 段）

int main()
{
    int var;                    // var 在栈区
    char *p_var;                // p_var 在栈区
    char arr[] = "abc";         // arr 为数组变量，存储在栈区；"abc"为字符串常量，存储在常量区
    char *p_var1 = "123456";    // p_var1 在栈区；"123456"为字符串常量，存储在常量区
    static int s_var = 0;       // s_var 为静态变量，存在静态存储区（.data 段）
    p_var = (char *)malloc(10); // 分配得来的 10 个字节的区域在堆区
    free(p_var);
    return 0;
}

关于堆与栈的区别：

1.申请方式：栈是系统自动分配的，堆是由程序员手动分配的

2.申请后系统的响应：栈如果空间不足，就会分配失败——栈溢出；申请堆空间的过程，则是在一个链表（记录了空闲地址空间的链表）上寻找第一个大于申请空间的节点分配给程序，将该节点从链表中删除。该空间的首地址通常存放本次分配空间的大小，以便释放。

3.栈在内存中是连续的一块空间，想低地址扩展；堆在内存中的空间是不连续的，向高地址扩展。

4.申请效率：栈是系统自动分配的，申请效率高，程序员无法控制；堆是程序员主动申请的，效率低，申请方便但容易产生碎片。

5.存放内容：栈中存放的是局部变量，函数参数；堆中存放的内容由程序员控制。

3. C++中的指针参数传递和引用参数传递

简单来说：

  • 指针参数传递的本质是值传递，即主调中的指针是p1，而函数中的指针是p2，只不过他们两个指向的是同一个地址。即多了一个替身；
  • 引用参数传递，则是局部变量直接存放了实参变量的地址。被调函数对形参的任何操作，都会影响到主调函数中的实参变量。

从编译的角度来看，程序编译的时候会将指针参数和引用参数都添加到符号表上，符号表上记录了参数的名字和地址，对于指针参数而言，记录的地址是指针变量的地址，并不是指针所指的地址；而对于引用而言，记录的地址是就是引用的对象的地址。

这导致最大的问题是，指针可以改变其指向的对象，而引用对象不可以修改。

1.什么是跨域问题？

浏览器从一个域名的网页去请求另一个域名的资源时，域名、端口、协议任一不同，都是跨域。

  1. 端口和协议的不同，只能通过后台来解决。
  2. 在跨域问题上，仅仅是通过“URL的首部”来识别而不会根据域名对应的IP地址是否相同来判断。“URL的首部”可以理解为“协议, 域名和端口必须匹配”。
  3. localhost和127.0.0.1虽然都指向本机，但也属于跨域。即便两个不同的域名指向同一个ip地址，也非同源。
  4. 跨域并不是请求发不出去，请求能发出去，服务端能收到请求并正常返回结果，只是结果被浏览器拦截了

解决方法：

  • JSONP：不需要过多了解

  • CORS：服务端设置 Access-Control-Allow-Origin 就可以开启 CORS。 该属性表示哪些域名可以访问资源，如果设置通配符则表示所有网站都可以访问资源。

  • nginx反向代理；

https://github.com/ljianshu/Blog/issues/55

2. 在项目里使用了nginx，说一下正向代理和反向代理的区别？

正向代理：内网client通过代理proxy访问外网server

  • 访问原来无法访问的资源，比如说使用VPN(virtual private network)访问google
  • 作为缓存，加速资源访问速度
  • 对内记录用户访问信息，对外隐藏用户记录
  • 对客户端访问授权，上网进行认证

反向代理：外网client的请求通过代理转发到内网server

  • 保证内网安全，阻止web攻击。大型网站，通常将反向代理作为公网访问地址，Web服务器是内网
  • 负载均衡，通过反向代理服务器来优化网站的负载。

简单来说，正向代理服务器是和客户端一体的；反向代理服务器是和服务器一体的。
