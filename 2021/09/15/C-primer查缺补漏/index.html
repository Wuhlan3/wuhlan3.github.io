类

  • 参数列表后的const——常量成员函数

    常量成员函数，可读取类中的数据成员，但不能修改。 一个常量成员函数，可以被 const 和 non-const 类对象调用； 而非常量成员函数，例如 AddYear()，则只能被 non-const 型类对象调用。

    class Date {
public:
    int GetYear() const { return y_; }int GetMonth() const { return m_; }
    int GetDay() const { return d_; }
    void AddYear(int n); // add n years
private:
    int y_, m_, d_;  
};

void f(Date& d, const Date& cd)
{
    int i = d.GetYear();    // OK
    d.AddYear(1);        // OK

    int j = cd.GetYear();    // OK
    cd.AddYear(1);        // error
}

  • class关键字和struct关键字。唯一的不同在于struct关键字，在定义第一个访问说明符之前的成员都是public；class关键字，在定义第一个访问说明符之前的成员都是private。所以为了统一编程风格，当希望所有成员都是public时，用struct；当希望有的成员为private时，用class。

  • 友元函数

    #include <iostream>
 
using namespace std;
 
class Box
{
   double width;
public:
   friend void printWidth( Box box );
   void setWidth( double wid );
};
 
// 成员函数定义
void Box::setWidth( double wid )
{
    width = wid;
}
 
// 请注意：printWidth() 不是任何类的成员函数
void printWidth( Box box )
{
   /* 因为 printWidth() 是 Box 的友元，它可以直接访问该类的任何成员 */
   cout << "Width of box : " << box.width <<endl;
}
 
// 程序的主函数
int main( )
{
   Box box;
 
   // 使用成员函数设置宽度
   box.setWidth(10.0);
   
   // 使用友元函数输出宽度
   printWidth( box );
 
   return 0;
}

    可以看到，友元函数printWidth是不是属于任何类的成员函数，但是他能够调用Box内的私有成员。友元这种东西本身和类的封装性是有一定的冲突的，因为友元相当于在一个封装好的类上面打开了一个让其他模块得以进入该类的通道。因此在面向对象编程中，友元是不推荐使用的东西。

  • 隐式内联函数与显式内联函数、重载成员函数

    class Screen{
    ...
    char get() const{
        return contents[cursor];
    }	//隐式内联函数
    inline char get(pos ht, pos wd) const;//显式内联函数
    ...
};

char Screen::get(pos r, pos c) const{
    pos row = r * width;
    return constents[row+c];
}

    只要是定义在类内部的成员函数是自动inline的，定义在类外部的成员函数需要显式声明（可以在类内加inline关键字，也可以在类外加inline关键字）。当然，这不意味着这个函数一定能内联处理，编译器会自动判断一个函数是否适合声明为内联函数，如果内联会造成代码膨胀，增大开销，则会将其转化为普通函数进行处理。

    重载成员函数：另外，这里定义了两个get函数，编译器会根据实参的数量来决定运行那个版本的函数。

  • 返回*this的成员函数

    class Screen{
    public:
        Screen &set(char);
        Screen &move(pos, pos);
}
inline Screen &Screen::move(pos r, pos c){
    ...
    return *this;
}
inline Screen &Screen::set(char c){
    ...
    return *this;
}

    对于这样的函数，可以这样使用：

    myScreen.move(4,0).set('#');
//效果等价于
myScreen.move(4,0);
myScreen.set('#');

    如果，move函数的返回值没有使用引用类型Screen &，则：

    myScreen.move(4,0).set('#');
//效果等价于,myScreen没有执行set函数
Screen temp = myScreen.move(4,0);
temp.set('#');
