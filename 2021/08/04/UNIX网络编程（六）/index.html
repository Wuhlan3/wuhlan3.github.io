UNIX网络编程（六）

本文最后更新于：2 years ago

I/O模型

  • 阻塞型I/O
  • 非阻塞型I/O
  • I/O复用
  • 信号驱动式I/O（SIGIO）
  • 异步I/O（POSIX的aio_系列函数）

阻塞型I/O

例子：本书把recvfrom当作是系统调用（不同操作系统可能不太一样），因为涉及到内核与应用进程的切换。在调用recvfrom到返回的这一段时间内，是被阻塞的。

非阻塞型I/O

特点用一句话概括。进程把套接字设置成非阻塞型是在通知内核：当所请求的I/O操作非得把本进程投入睡眠才能完成时，不要投入睡眠而是返回一个错误。

如图，当一个应用进程像这样对一个非阻塞描述符循环调用recvfrom，称为轮询（polling）。应用进程持续访问内核，查看内核是否就绪。这样会造成大量的CPU时间损耗，值在一小部分系统中会应用到。

I/O复用

调用select或poll，阻塞在这两个系统调用的某个之上。

注意：相比于图6-1，该方法使用了两次系统调用（select，recvfrom）

信号驱动式I/O

其优势在于，等待数据报到达之前，不需要阻塞。

异步I/O

I/O复用的场合

  1. 当客户使用多个描述符时（套接字和交互式输入），必须使用I/O复用。
  2. 一个用户同时处理多个套接字
  3. 一个服务器既要处理listen套接字，又要处理connect套接字
  4. 一个服务器既要处理TCP，又要处理UDP
  5. 一个服务器处理多个服务或者协议

select函数

调用select告知内核对哪些描述符（读、写或异常条件）感兴趣以及等待多长时间。事实上任何描述符都可以使用select来进行测试。

#include<sys/time.h>

    //返回，若有就绪描述符则返回数目，若超时则返回0，若出错则返回-1

关于fd_set

要怎么来表示多个描述符呢？

定义了一种数据类型fd_set和四个宏，具体实现不需了解：

void FD_ZERO(fd_set *fdset);			/*将描述符集清零*/
void FD_SET(int fd, fd_set *fdset);		/*将其中一个描述符位  置1*/
void FD_CLR(int fd, fd_set *fdset);		/*将其中一个描述符位  置0*/
void FD_ISSET(int fd, fd_set *fdset);	/*判断某一位是否被设置*/

使用方法：

fd_set 	socket_set;
FD_SET(1, &socket_set);
FD_SET(4, &socket_set);

注意，描述符集参数是 值-结果参数。原本为1的位，调用完后可能为0；

此外，maxfdp1也容易忘记+1。是在最大描述符的基础上+1

关于struct timeval

其中，struct timeval 的结构如下，指定了这段时间的秒数和微秒数：

struct timeval{
    long tv_sec;
    long tv_usec;
};

这个参数有三种可能：

  1. 永远等待下去：指针设置为空
  2. 等待一段固定时间：设置秒数和微秒数
  3. 根本不等待：检查描述符后里面返回，这就是之前提到的轮询。把秒数和微秒数设置为0。

就绪条件

对str_cli进行修改

之前我们遇到的问题是：客户端进程阻塞在fgets，不能及时读取socket描述符的信息。下面来进行改进：

  1. 如果对端发送数据，那么该套接字变为可读，返回一个大于0的值（读入字节数）
  2. 如果对端发送一个FIN，那么该套接字变为可读，read返回一个0（EOF）
  3. 如果对端发送一个RST（对端主机崩溃或重新启动），那么该套接字变为可读，read返回一个-1（errno含有错误码）
#include "unp.h"
void str_cli(FILE *fp, int sockfd){
    int maxfdp1;	//最大描述符数
    fd_set rset;	//read set 读入的描述符集
    char sendline[MAXLINE], recvline[MAXLINE];
    FD_ZERO(&rset);	//初始化，清零
    
    for( ; ; ) {
    	FD_SET(fileno(fp), &rset);	//设置文件描述符。其中fileno可以把文件指针转为描述符
    	FD_SET(sockfd, &rset);		//设置套接字描述符
    	maxfdp1 = max(fileno(fp), sockfd)+1;	// +1 +1 +1
    	Select(maxfdp1, &rset, NULL, NULL, NULL);
    	
    	if(FD_ISSET(sockfd, &rset)){	//若读套接字已就绪
    	    if(Readline(sockfd, recvline, MAXLINE) == 0 ){		//EOF
    	    	err_quit("str_cli:server terminated prematurely");
    	    }
    	    Fputs(recvline, stdout);
    	}
    	if(FD_ISSET(fileno(fp), &rset)){	//若读文件流已就绪
    	    if(Fgets(sendline, MAXLINE, fp) == NULL ){
    	    	return;
    	    }
    	    Writen(sockfd, sendline,strlen(sendline) );
    	}
    }
}

shutdown函数

一般来说关闭连接使用close函数，但是有两个限制：

  1. close吧描述符的counter-1，只有counter为0时才关闭套机字
  2. close终止读和写两个方向上的数据传送

这就是另一个需要解决的问题：有的时候客户端发送完数据后，直接就发送FIN了，是要告知服务器我已经发送完数据了。此时客户端仍想要保持套接字打开，继续接收读取服务器发回来的数据。这就要用到shutdown函数

#include<sys/socket.h>
int shutdown(int sockfd, int howto);
/*
	SHUT_RD:关闭连接读半部
	SHUT_WR:关闭连接写半部，对于TCP而言，这就是半关闭状态half-close
	SHUT_RDWR:关闭连接读写。等价于调用分别上面两次shutdown
*/

str_cli的再修改

#include "unp.h"
void str_cli(FILE *fp, int sockfd){
    int stdineof = 0; //判断是否收到eof，只要标志为0，就select标准输入的可读性
    int maxfdp1;
    fd_set rset;
    char buf[MAXLINE];
    int n;
    FD_ZERO(&rset);
    for( ; ; ) {
    	if(stdineof == 0)//若没收到，则开启该描述符
    	    FD_SET(fileno(fp), &rset);
    	FD_SET(sockfd, &rset);
    	maxfdp1 = max(fileno(fp), sockfd)+1;
    	Select(maxfdp1, &rset, NULL, NULL, NULL);
    	//连接输入（接收）
    	if(FD_ISSET(sockfd, &rset)){
    	    if( (n == Read(sockfd, buf, MAXLINE)) == 0){
    	    	if(stdineof == 1)//若收到服务器发来的FIN，且自己在这之前已经关闭则返回
    	    	    return;
    	    	else				//否则报：“服务器提前结束”
    	    	    err_quit("str_cli:server terminated prematurely");
    	    }
    	    Write(fileno(stdout), buf, n);
    	}
        //用户输入
    	if(FD_ISSET(fileno(fp), &rset)){
    	    if( (n = Read(fileno(fp), buf, MAXLINE)) == 0){
    	    	stdineof = 1;//若用户写入EOF，则置为1
    	    	Shutdown(sockfd, SHUT_WR);//关闭连接的写半部
    	    	FD_CLR(fileno(fp), &rset);//将用户输入的读描述集也关闭
    	    	continue;
    	    }
    	    Writen(sockfd, buf, n);
    	}
    }
}

使用selcet实现服务器

使用select函数可以替换掉每次fork一个子进程。实现过程与上面客户端的str_cli类似。不再赘述。

pselect函数

#include <sys/select.h>
#include <signal.h>
#include <time.h>
int pselect(int maxfdp1, fd_set *readset, fd_set *writeset, fd_set *exceptest, const struct timespec *timeout, const sigset_t *sigmask);

pselect与select仅有以下几点区别

  • timeout的结构变为秒和纳秒

  • select的timeout是值-结果参数，pselect的timeout不会改变

  • 多了一个信号掩码参数。

poll函数

#include <poll.h>
int poll(struct pollfd fd[], nfds_t nfds, int timeout);

struct pollfd{
　　int fd;          //文件描述符
　　short events;    //请求的事件
　　short revents;   //返回的事件
　　};
常量 说明
POLLIN 普通或优先级带数据可读
POLLRDNORM 普通数据可读
POLLRDBAND 优先级带数据可读
POLLPRI 高优先级数据可读
POLLOUT 普通数据可写
POLLWRNORM 普通数据可写
POLLWRBAND 优先级带数据可写
POLLERR 发生错误
POLLHUP 发生挂起
POLLNVAL 描述字不是一个打开的文件

注意：后三个只能作为描述字的返回结果存储在revents中，而不能作为测试条件用于events中

timeout值 说明
INFTIM 永远等待
0 立即返回，不阻塞进程
＞0 等待指定数目的毫秒数
