UNIX网络编程（九）

本文最后更新于：2 years ago

守护进程

守护进程（daemon）：是在后台运行且不与任何控制终端关联的进程

守护进程的启动方法：

  1. 系统启动时，由系统初始化脚本启动。一般在/etc目录下，或者以/etc/rc开头的目录

  2. 许多网络服务器由inetd超级服务器启动

  3. cron守护进程按规则定期执行一些程序

  4. 用at命令指定在将来的某一时刻执行程序。

  5. 还可以从用户终端、前台、后台启动

syslogd守护进程

启动过程：

  1. 读入配置文件，通常是/etc/syslog.conf。设定守护进程对接收每次键入的各种等级消息的处理。

  2. 创建一个unix域套接口，捆绑路径名/var/run/log

  3. 创建一个UDP套接口，给它捆绑端口514（syslog服务专用的端口号）

  4. 打开路径名/dev/klog，内核中的所有出错消息作为这个设备的输入出现

在这之后，syslogd守护进程就会在后台无限循环地运行。可以调用select来等待它的三个描述符（分别对应步骤2，3，4）

syslog函数：syslog常被称为系统日志或系统记录，是一种用来在互联网协议（TCP/IP）的网上中传递记录档消息的标准。

void syslog(int priority,const char * message,...); 
//priority是level和facility两者地结合

举例：假设rename函数调用失败

syslog(LOG_INFO|LOG_LOCAL2,"rename(%s,%s):%m",file1,file2);

%m:将当前的errno值的出错消息取代

daemon_init函数

通过调用它，可以将一个普通进程变为守护进程

#include "unp.h"
#include <syslog.h>
#define MAXFD 64
extern int daemon_proc;
void daemon_init(const char * pname,int facility){
int i;
pid_t pid;
 
if((pid = Fork())!=0) //调用fork，终止父进程，留下子进程继续运行。
    exit(0);
 
setid(); //创建一个新的登陆会话session;当前进程变为新会话的会话头进程以及新进程组的头进程，从而不再有控制终端
Signal(SIGHUP,SIG_IGN); //忽略SGIHUP信号并再次调用fork
 
if((pid=Fork())!=0) //确保将来即使打开一个终端设备，也不会自动获得控制终端。
    exit(0);
 
daemon_proc = 1; //非零时，出错处理函数将调用syslog函数取代fprintf输出到标准错误输出
 
chdir("/");
umask(0);//将工作目录改到根目录
 
for(i=0;i<MAXFD;i++) //关闭守护进程从运行开始打开的所有文件描述字
    close(i);
 
openlog(pname,LOG_PID,facility);
}

inetd守护进程

/etc/inetd.conf配置超级服务器处理的服务

守护进程工作流程：

1 启动读/etc/inetd.conf文件，指定创建相应类型的套接口

2 调用bind，绑定众所周知的端口和通配地址

3 对TCP调用listen

4 调用select等待套接口变为可读

5 返回一个可读的套接口，如果是TCP套接口，就调用accept接受

6 调用fork，由子进程处理服务请求

小结

守护进程是在后台运行且不与任何控制终端关联的进程。守护进程产生的所有输出通过调用syslog函数发送给syslogd守护进程。

daemon_init函数把一个进程作为守护进程启动，经过以下步骤：调用fork到后台运行；调用setsid建立一个新的POSIX会话并成为会话头进程；fork以避免获得控制终端；改变工作目录和文件创建模式掩码，关闭非必要描述符。
